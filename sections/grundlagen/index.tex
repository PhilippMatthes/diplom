\chapter{Grundlagen}\label{ch:grundlagen}

In diesem Kapitel sollen die zentralen Motivationen und Ideen hinter einer Verkehrsmittelerkennung auf Smartphones diskutiert werden. Das Kapitel untergliedert sich in drei aufeinander aufbauende Sektionen. Zunächst wird eruiert, wie die zugrundeliegenden Daten erfasst werden, welche Probleme hieraus entstehen und wie die Daten für eine Klassifikation vorbereitet werden können. Anschließend werden Machine-Learning-Ansätze diskutiert, welche zur Verkehrsmittelklassifikation genutzt werden können. Zum Schluss wird gezeigt, welche Konzepte und Ideen hinter einer Portierung von Machine-Learning-Ansätzen auf Smartphones stehen.

\section{Zusammenfassung}

Durch Smartphones können verschiedene Aktivitätsdaten aufgezeichnet werden. Die dabei zum Einsatz kommenden Sensorsysteme sind durch ihre eigenen Fehlermuster geprägt, bei der Positionsapproximation über \acrshort{gnss} ist zusätzlich der sehr hohe Energieverbrauch problematisch. Es wurden verschiedene Methoden diskutiert, welche genutzt werden können, um die inhärenten Fehlermuster zu mitigieren. Zum einen wurde die sensitive Nutzung der Positionsbestimmung durch Konfiguration der benötigten Präzision motiviert, um auf energiesparendere Triangulationsmechanismen über Funknetze auszuweichen. Zum anderen wurden verschiedene mathematische Verfahren zur Reduktion von Rauschen, Rekonstruktion im Weltkoordinatensystem, Transformation der Zeitliniendaten in Frequenzdaten, Skalierung und Approximation vorgestellt. Speziell das \acrshort{ahrs}-Verfahren bietet eine wichtige Grundlage für die Signalvorverarbeitung. Außerdem wurde beschrieben, wie Aktivitätsdaten interpoliert werden können, um die Datenpunkte in ein festes Abtastraster zu bringen.

Anschließend wurden Machine-Learning-Modelle als Möglichkeit diskutiert, die vorverarbeiteten Daten zu klassifizieren. Dazu können die Daten zunächst segmentiert und dann in Features überführt werden, welche als Eingabe für das Machine-Learning-Modell dienen. Hierbei können Shallow Features und Non-Shallow Features berechnet und kombiniert werden. Für ein besseres Verständnis der Domäne wurde das Problem der Verkehrsmittelklassifikation in klassische Machine-Learning-Probleme eingeordnet. Außerdem wurde motiviert, warum eine Verkehrsmittelklassifikation hervorragend über Machine-Learning-Modelle realisiert werden kann. In diesem Rahmen wurden die drei Grundmethoden für das Training von Machine-Learning-Modellen, insbesondere das Supervised Learning, beschrieben. Als Grundlage für das Supervised Learning von größeren Machine-Learning-Modellen wurde das Gradientenabstiegsverfahren erklärt und auf wichtige Probleme wie das Overfitting und das Underfitting hingewiesen. Um diese Probleme zu vermeiden, wurden zentrale Ideen zur Modellvalidierung und -regularisierung eruiert. Da die bestehenden Movebis-Arbeiten \cite{matusek_anwendung_2019}, \cite{werner_kontinuierliche_2020} und \cite{stojanov_continuous_2020} künstliche neuronale Netzwerke als Machine-Learning-Modell verwenden, wurden deren Architekturen und Funktionsweisen im Überblick gezeigt. Die Grundlage dieser Architekturen stellen spezielle Neuronen und Aktivierungsfunktionen dar.

Mithilfe vorverarbeiteter Daten und der gezeigten Machine-Learning-Modelle kann eine Verkehrsmittelklassifikation implementiert werden, dies zeigen \cite{matusek_anwendung_2019}, \cite{werner_kontinuierliche_2020} und \cite{stojanov_continuous_2020}. In diesen Arbeiten wird allerdings nicht beschrieben, wie die Modelle auf Smartphones portiert werden können. Der dritte Teil des Grundlagenkapitels diskutiert daher die fundamentalen Ideen der Modellportierung auf Smartphones. Hierzu wurde gezeigt, dass sich Smartphones als Edge-Geräte interpretieren lassen und somit die Konzepte des Forschungsbereichs Edge AI auf diesen speziellen Anwendungsfall erweitert werden können. Hierzu wurden verschiedene Deployment-Modelle und die daraus resultierenden Konsequenzen diskutiert. Es wurden erweiterte Trainingskonzepte wie das Federated Learning identifiziert, welche für einen All-In-Edge Ansatz genutzt werden können. Als limitierende Faktoren von Edge-Geräten wurden Hardware- und Software-Limitationen beschrieben. In diesem Kontext wurden Ideen samt Vor- und Nachteilen vorgestellt, mithilfe derer eine Modelloptimierung durchgeführt werden kann. Abschließend wurde gezeigt, wie über Profiling die Modellgüte dem Ressourcenverbrauch gegenübergestellt werden kann.
